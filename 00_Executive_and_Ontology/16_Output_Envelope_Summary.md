# Output Envelope Summary

## Local definitions

Amazing Game Engine [AGE] means the complete platform described by this archive.

Corpus Arbitration Layer [CAL] means the component that answers unstructured questions against LMS-Graph.

## Purpose

AGE output should be readable to players and auditable to systems. The archive therefore distinguishes narrative output from structured envelopes.

## Runtime outcome envelope

A runtime outcome should include:

- User input.
- Interpreted intent.
- Action Candidate ID.
- Entity versions read.
- Validation result.
- Domain modules invoked.
- Random seed or deterministic resolver reference.
- State deltas committed.
- Events published.
- TickPolicy applied.
- Context packet ID.
- Fiction Constructor output.
- Output Verifier result.
- Replay ID.

## CAL arbitration envelope

A CAL answer should include:

- User question.
- Interpreted query.
- Corpus partition searched.
- Sources retrieved.
- Authority tier.
- Version.
- Scope or jurisdiction.
- Confidence.
- Conflicts found.
- Primary answer.
- Alternative readings.
- Human decision point.
- Deep-dive paths.
- Unsupported or excluded material.
- Override/correction field.

## Role output envelope

A role response should include:

- Role ID.
- Role Semantic Contract version.
- Role knowledge scope.
- Evidence requirements.
- Behavioral constraints applied.
- Memory scope used.
- CAL query references, if any.
- Output verification result.
- Drift or violation flags.
