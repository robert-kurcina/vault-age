# Engine Fidelity Summary

## Local definitions

Large Language Model [LLM] means a generative language model used for language interpretation or presentation.

Large Model System [LMS] means the wider model, retrieval, graph, tool, agent, and workspace system around one or more LLMs.

Large Model System Graph [LMS-Graph] means the maintained graph/relational corpus substrate.

Corpus Arbitration Layer [CAL] means the component that answers unstructured questions against LMS-Graph.

Quality Assurance [QA] means testing and review; Semantic QA tests meaning, authority, role behavior, rules, state, and prose-state agreement.

User Experience [UX] means the practical user experience of authoring, play, review, and operation.

The guidance pass preserves the prior engine fidelity work and adds a clearer corpus-arbitration spine.

The preserved engine concepts are: state-authoritative runtime, LLM-as-Fiction-Constructor, Bridge Layer, Input Coordinator, State Mutation Engine, Event Bus, State Assembler, Output Verifier, partition topology, Memory Substrate, locale-to-world transitions, TickPolicy, troupe isolation, lazy ontology, concern overlays, Technology Level vectors, AGEScript, plot rebinding, Event Generator tables, Role Service, Role Semantic Contracts, Semantic QA, authoring UX, replay, and later Agent Service.

The strengthened corpus concepts are: LMS as Large Model System, LMS-Graph as graph/relational corpus substrate, CAL as Corpus Arbitration Layer, Rules Service as the first CAL deployment, answer envelopes, authority tiers, version-aware sources, conflict surfacing, human decision points, and market-selected expansion.

The project remains game-first. The first corpus is a game rules corpus because it is bounded, testable, high-volume, conflict-rich, and safe. The same architecture can later apply to other corpora, but the game runtime and Rules Service are the correct proving ground.
