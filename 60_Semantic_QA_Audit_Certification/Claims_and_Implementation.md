# Claims and Implementation — Semantic Quality Assurance [QA], Audit, and Certification Artifacts

## Local definitions

Quality Assurance [QA] means testing and review; Semantic QA tests meaning, authority, role behavior, rules, state, and prose-state agreement.

## Claims

- Output must be checked against state.
- Roles must be tested against contracts.
- Audit records must support replay and review.

## Implementation steps

1. Define verifier checks.
2. Define semantic QA rubrics.
3. Log execution artifacts.
4. Add override records.
5. Build audit package export.
6. Add benchmark reports.

## Risks and controls

- Risk: false confidence. Control: describe tests precisely.
- Risk: audit cost. Control: tiered logging.
- Risk: premature certification claims. Control: artifact language, not institutional guarantees.
