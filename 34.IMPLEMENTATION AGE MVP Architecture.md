# **Key MVP Principles Illustrated**

1. **Troupe Isolation**
    
    - Each troupe operates in its **own world-state sandbox**.
        
    - No cross-troupe interaction for MVP → avoids concurrency conflicts.
        
2. **Chronological Partitions**
    
    - Narrative + spatial partitions scale with abstraction:  
        Epic → Arc → Chapter → Act → Scene → Spotlight
        
    - Tick duration maps to abstraction layer (minutes → years).
        
3. **MoE Coordination**
    
    - Input Coordinator delegates **player actions** to the correct **utility modules**.
        
    - Aggregates responses and updates world-state.
        
4. **Utilities / Non-LLM Modules**
    
    - Resolve **deterministic computations** (tests, movement, resource updates).
        
    - Produce **flags** for LLM-driven narrative.
        
5. **NPC / Background Tick**
    
    - Simulates **off-screen events**, updates world-state asynchronously.
        
    - Flags propagate up and down partitions, providing hooks for LLM story generation.
        
6. **Fiction Constructor**
    
    - Reads **world-state + flags** relevant to the current narrative partition.
        
    - Generates **player-facing narrative**.
        
    - Updates hooks and flags for future actions/events.
        
7. **Player Output**
    
    - Receives narrative → makes next moves → **Input Coordinator** loops again.

# **How Flag Propagation Works**

1. **Player-Triggered Flags**
    
    - Player moves through a submap → sets local flags (e.g., `"door_open": true`, `"npcA_spotted": true`)
        
    - Flags propagate **up** to higher narrative/spatial partitions as needed (Act → Chapter → Arc)
        
    - Influences subsequent story hooks and NPC behavior
        
2. **Background NPC Flags**
    
    - NPCs act autonomously in submaps, loci, or regions
        
    - Flags capture state changes (e.g., `"village_attacked": true`, `"artifact_stolen": true`)
        
    - Propagate upward to influence broader narrative context
        
3. **Flag Types**
    
    - **Local:** confined to scene/locus/submap
        
    - **Intermediate:** act/chapter level
        
    - **Global:** arc/epic level, affecting future events
        
    - **Temporal metadata:** indicates chronological layer (Epic → Spotlight)
        
4. **LLM Narrative Generation**
    
    - Reads all flags in the **current partition context**
        
    - Generates player-facing story dynamically
        
    - Sets hooks/flags for **future events**, keeping causality coherent
        
5. **Result**
    
    - Emergent story without requiring continuous recursion
        
    - Lower partitions influence higher partitions (causality up)
        
    - Upper partitions set constraints for lower partitions (guidance down)
        
    - Background NPC activity and player actions coexist seamlessly
