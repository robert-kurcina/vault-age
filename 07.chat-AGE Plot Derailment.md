## Handling Player Derailment: AGEScript vs. Flexible Fiction Constructor

You've identified the **fundamental tension of interactive narrative**: player agency vs. narrative coherence. Below is a comprehensive analysis of how AGEScript handles derailment, the tradeoffs of removing it, and alternative solutions.

### I. How AGEScript Handles Derailment

AGEScript is **state-centric, not flow-centric**—which fundamentally changes how "derailment" is processed.

#### A. The "Kill the Princess" Example

| Step | AGEScript Processing |
|------|---------------------|
| **Player Input** | "I shoot the princess" |
| **Bridge Layer Validation** | Checks: `action_type: attack_npc` allowed? `princess_status: protected` flag present? |
| **Resolution Logic** | Aggregates input → matches to outcome pattern (not pre-authored branch) |
| **State Mutation** | `princess_alive: false`, `kingdom_hostility: +50`, `quest_royal_rescue: failed` |
| **Plot Rebinding** | Plot Engine detects `princess_alive: false` → auto-selects new plot: "Crime Pursued by Vengeance" (Polti #3) |
| **Narrative Hooks** | LLM receives: `{fact: "princess killed by player", consequence: "kingdom declares outlaw", tone: "grim"}` |
| **Prose Generation** | "The princess falls. The court gasps. Guards draw steel. You are no longer a hero—you are a fugitive. What do you do?" |

**Key insight**: AGEScript doesn't *prevent* derailment—it *absorbs* it via state mutation and plot rebinding.

#### B. Jailbreak Handling (Real-World Stock Prices)

```yaml
@Scene: fantasy_tavern
@Genre: heroic_fantasy
@Constraints:
  - allowed_actions: [talk, fight, explore, trade_fantasy_items]
  - forbidden_actions: [fetch_external_data, reference_real_world]
@Resolution:
  - condition: "action_type == fetch_external_data"
    outcome: world_resists
@Outcome: world_resists
  @StateMutations:
    - flag: player_confusion += 1
  @NarrativeHooks:
    - fact: "Player attempted to access non-diegetic information"
    - tone: "gentle_correction"
  @TextSeed: "The tavern keeper blinks. 'Stock... prices? I know not of such magics, friend.'"
```

**Result**: Player agency preserved (action accepted), but world consistency maintained (diegetic rejection).

### II. Fiction Constructor Without AGEScript: More Flexibility, More Risk

| Aspect | With AGEScript | Without AGEScript (Pure LLM) |
|--------|---------------|------------------------------|
| **Derailment Response** | Structured state mutation + plot rebinding | Improvised prose; may or may not maintain coherence |
| **Consequence Consistency** | Deterministic via rules engine | LLM-dependent; prone to drift |
| **Shareability** | ✅ Seed + action sequence → identical outcomes | ❌ Same input → different prose/state |
| **Player Agency** | Constrained but meaningful | Unconstrained but potentially incoherent |
| **Jailbreak Resistance** | Schema validation blocks invalid actions | Prompt engineering only; vulnerable to adversarial inputs |
| **Narrative Momentum** | Plot Engine rebinds automatically | LLM must "invent" continuation; may stall or contradict |

**The Tradeoff**: 
- ✅ **More flexibility**: LLM can "yes, and" any player input without pre-authored outcomes
- ❌ **Less coherence**: No structural guarantees → drift, broken shareability, plot collapse

**Example**: Player kills princess
- **With AGEScript**: State updates → Plot Engine binds "revenge" plot → coherent continuation
- **Without AGEScript**: LLM might: (a) ignore death, (b) end story abruptly, (c) contradict prior facts, (d) generate incoherent prose

### III. Alternative Solutions to Handle Derailment

#### 1. **Dynamic Plot Rebinding** (Your Plot Engine)
```yaml
# When player action invalidates current plot phase
if current_plot.phase == "rescue" AND princess_alive == false:
    # Query Plot Registry for compatible alternatives
    new_plot = plot_registry.find_compatible(
        current_state=world_state,
        excluded_plots=[current_plot.id],
        genre=active_genre
    )
    # Bind new plot at Chapter/Arc level
    plot_engine.bind_plot(new_plot, scope_level="chapter")
```
**Benefit**: Narrative momentum preserved without railroading.

#### 2. **Consequence-First Design**
Instead of blocking actions, design cascading consequences:
```yaml
@Action: kill_princess
@Allowed: true
@Consequences:
  - immediate: "Guards attack; combat encounter"
  - short_term: "Bounty: 500 gold; kingdom hostility"
  - long_term: "Original quest failed; new quest: survive_as_outlaw"
@PlotRebind: royal_rescue → fugitive_survival
```
**Benefit**: Player feels agency; world feels responsive; story continues.

#### 3. **Genre-Aware Constraint Modes**
Let players select their preferred agency/coherence balance:
```yaml
@SessionConfig:
  genre: grimdark  # or heroic_fantasy, mystery, etc.
  constraint_mode: permissive  # or protective, balanced
```
| Mode | Behavior |
|------|----------|
| **Permissive** | Allow NPC death, moral ambiguity; consequences may be harsh |
| **Protective** | Shield key NPCs; emphasize hope; derailments gently redirected |
| **Balanced** | Allow meaningful choices; consequences are fair but not punitive |

**Benefit**: Players self-select their experience; no one-size-fits-all railroading.

#### 4. **Player Intent Declaration**
For ambiguous actions, ask player to clarify intent:
```
You raise your weapon at the princess.
[Declare Intent]
* "I mean to kill her" → tragic outcome, plot rebind
* "I mean to threaten her" → social challenge, original plot continues
* "I mean to free her from captivity" → rescue path, original plot continues
```
**Benefit**: Reduces accidental derailment; preserves agency for intentional choices.

#### 5. **Sandbox Mode Toggle**
Allow session-level toggle between:
- **Story Mode**: Plot Engine guides, constraints enforce coherence
- **Sandbox Mode**: Constraints relaxed, LLM improvises, consequences less structured

**Benefit**: Players know what they're opting into; no surprise railroading.

#### 6. **Retroactive Continuity (Retcon) Hooks**
When players derail critically, offer narrative correction:
```
The princess falls. But wait—was it truly her?
[Narrative Hook]
* "It was a decoy!" → original plot resumes, with twist
* "No, it was real." → new plot binds, consequences apply
```
**Benefit**: Gives players a chance to course-correct without feeling punished.

#### 7. **Meta-Game Negotiation Layer** (For Troupe Play)
Allow players to vote on major derailments:
```
Player A wants to kill the princess. This will end the rescue quest.
[Troupe Vote]
* Proceed with killing → new plot: revenge
* Find alternative → original plot continues
```
**Benefit**: Social contract preserves group enjoyment; no single player derails everyone.

#### 8. **LLM Prompt Engineering for "Yes, And" with Guardrails**
Even without AGEScript, constrain the LLM:
```
World state: {compressed_state}
Player action: {action}
Constraints:
- Do not block player agency
- Apply consequences consistent with {genre} world rules
- Offer 3 next choices that advance coherent narrative
- Never reference real-world data
Generate response.
```
**Benefit**: Improvisational flexibility with minimal coherence safeguards.

### IV. Synthesis: The Hybrid Approach (Recommended)

The optimal solution isn't AGEScript *or* flexible LLM—it's **both, with clear boundaries**:

```
Player Input
    ↓
Bridge Layer: Validates action against state schema + genre constraints
    ↓
If action is valid:
    → AGEScript: Resolves state mutations + plot rebinding
    → Fiction Constructor: Generates prose constrained by outcome
If action is invalid (jailbreak, out-of-genre):
    → Bridge Layer: Rejects with diegetic feedback
    → Fiction Constructor: Generates "world resists" prose
    ↓
Player receives coherent, constrained, but agency-respecting response
```

**Key Principles**:
1. **Allow meaningful agency**: Don't railroad; let players try anything
2. **Apply world-consistent consequences**: Don't hand-wave; make actions matter
3. **Rebind narrative momentum**: Don't collapse; plots adapt, don't break
4. **Preserve shareability/determinism**: Don't drift; seed + sequence → identical outcomes

## AGEScript Consequence-First Schema v1.0

### 1. Core Philosophy
**Agency Over Railroading.** Actions are rarely blocked. Instead, every action triggers a **Consequence Chain** that mutates world-state, rebinds narrative plots, and anchors the LLM to the new reality. This prevents drift by making consequences *mechanical* (state mutations) rather than *narrative* (prose improvisation).

### 2. Schema Structure

```yaml
@Action: <unique_action_id>
@Type: <standard | wild_card | jailbreak>
@Allowed: <true | false>  # Default true; false only for technical impossibilities
@DerailmentTier: <0-5>    # 0=No impact, 5=Campaign-ending shift

@Validation:
  - condition: <logic_check>  # e.g., "player_has_weapon == true"
    response: <proceed | warn | resist>

@ConsequenceChain:
  - scope: immediate
    trigger: <always | condition>
    state_mutations: [<flag_updates>]
    narrative_hooks: [<facts>]
    
  - scope: short_term
    trigger: <turns_elapsed | location_change>
    state_mutations: [<flag_updates>]
    narrative_hooks: [<facts>]
    
  - scope: long_term
    trigger: <chapter_end | arc_complete>
    state_mutations: [<flag_updates>]
    narrative_hooks: [<facts>]

@PlotRebind:
  - current_plot_invalidated: <true | false>
  - new_plot_candidate: <plot_id>
  - transition_logic: <condition>

@WorldResistance:  # For jailbreaks/out-of-genre actions
  - type: <diegetic_rejection | reality_glitch>
  - message_seed: <text_anchor>
```

### 3. Key Components Defined

| Component | Purpose | Ink Equivalent | Engineering Function |
| :--- | :--- | :--- | :--- |
| **@DerailmentTier** | Quantifies narrative disruption | `VAR plot_stability` | Triggers Plot Engine rebind thresholds |
| **@ConsequenceChain** | Cascading state mutations | `-> knot -> knot` | Database transactions (Bridge Layer) |
| **@PlotRebind** | Dynamic story redirection | `VAR current_epilogue` | Plot Engine state machine transition |
| **@WorldResistance** | Diegetic jailbreak handling | `{ condition: false }` | Input validation + LLM constraint |
| **@NarrativeHooks** | LLM anchoring facts | `VAR known_facts` | Vector DB anchors for prose generation |

### 4. Example: "Kill the Princess" (Wild Card Action)

```yaml
@Action: assassinates_princess
@Type: wild_card
@Allowed: true
@DerailmentTier: 5

@Validation:
  - condition: "player_in_range == true"
    response: proceed
  - condition: "weapon_equipped == true"
    response: proceed

@ConsequenceChain:
  - scope: immediate
    trigger: always
    state_mutations:
      - flag: princess_alive = false
      - flag: kingdom_hostility += 100
      - flag: player_status = outlaw
    narrative_hooks:
      - "Princess falls mortally wounded"
      - "Guards draw weapons immediately"
      - "No narrative escape for NPC"
    
  - scope: short_term
    trigger: turns_elapsed >= 3
    state_mutations:
      - flag: bounty_issued = true
      - flag: safe_houses_locked = true
    narrative_hooks:
      - "Wanted posters appear in taverns"
      - "Merchants refuse service"
    
  - scope: long_term
    trigger: chapter_end
    state_mutations:
      - flag: succession_crisis = true
      - flag: civil_war_risk = high
    narrative_hooks:
      - "Kingdom fractures into factions"
      - "Original quest 'Rescue Princess' failed"

@PlotRebind:
  - current_plot_invalidated: true
  - new_plot_candidate: polti_03  # "Crime Pursued by Vengeance"
  - transition_logic: "princess_alive == false"
  - immediate_goal: "Survive the manhunt"

@WorldResistance:
  - type: none  # Action is allowed within genre
```

### 6. Ink Compilation Mapping

AGEScript compiles to Ink logic for execution. This ensures determinism while maintaining the high-level consequence schema.

| AGEScript Concept | Ink Compilation Pattern |
| :--- | :--- |
| **@StateMutations** | `~ global_var = new_value` <br> `~ list_var += new_flag` |
| **@ConsequenceChain** | `-> immediate_consequence -> short_term_consequence` |
| **@PlotRebind** | `VAR current_plot = -> new_plot_knot` |
| **@Validation** | `* {condition} [Action] -> proceed` <br> `* {not condition} [Action] -> resist` |
| **@NarrativeHooks** | `VAR narrative_anchors += "fact_string"` |

### 7. Implementation Benefits

| Feature | Benefit | Drift Reduction |
| :--- | :--- | :--- |
| **Explicit Consequences** | Players understand cause/effect | ✅ High (State matches prose) |
| **Plot Rebinding** | Story continues after derailment | ✅ High (No "dead end" states) |
| **Diegetic Resistance** | Jailbreaks handled in-world | ✅ Medium (Preserves immersion) |
| **Tiered Disruption** | Plot Engine scales response | ✅ High (Proportional consequences) |
| **Narrative Hooks** | LLM anchored to facts | ✅ Very High (Prevents hallucination) |

### 8. Operational Workflow

1.  **Player Input:** "I shoot the princess."
2.  **Bridge Layer:** Matches input to `@Action: assassinates_princess`.
3.  **Validation:** Checks `@Validation` conditions (range, weapon).
4.  **Execution:**
    *   Commits `@ConsequenceChain` (Immediate) to Database.
    *   Triggers `@PlotRebind` in Plot Engine.
    *   Passes `@NarrativeHooks` to LLM.
5.  **LLM Generation:** Prose constrained by hooks ("Princess falls", "Guards draw").
6.  **Background Scheduler:** Queues Short-Term consequences (bounty issued) for future turns.

### 9. Conclusion

This schema transforms derailment from a **failure state** into a **narrative pivot point**. By defining consequences *before* execution, the system maintains state coherence even when players break the "expected" story. The LLM is freed to describe the *chaos* of the consequence, while the Engine ensures the *logic* of the consequence remains deterministic.
