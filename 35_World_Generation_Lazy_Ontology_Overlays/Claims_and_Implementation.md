# Claims and Implementation — World Generation, Lazy Ontology, and Overlays

## Claims

- Worlds begin as scaffolds and actualize detail on demand.
- Actualized detail becomes canonical.
- Overlays propose; Bridge Layer commits.

## Implementation steps

1. Define scaffold schemas.
2. Define actualization triggers.
3. Define canonicalization rules.
4. Build overlay proposal schema.
5. Implement TL vector model.
6. Add propagation locks for high-impact technologies.

## Risks and controls

- Risk: contradiction with scaffold. Control: validation before actualization.
- Risk: hidden simulator overlays. Control: proposal-only interfaces.
- Risk: TL incoherence. Control: vector TL and propagation locks.
