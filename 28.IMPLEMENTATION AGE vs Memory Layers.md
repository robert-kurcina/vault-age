## What each company is doing (and how they differentiate)

### Mem0

**What it’s doing:** A general-purpose “memory layer” for AI apps—captures and retrieves user interactions and preferences so agents feel persistent. It positions itself as universal and “self-improving,” and describes a **hybrid datastore** approach (graph + vector + key/value).   
**Differentiation:** Emphasis on _productized memory as a service_ + hybrid storage architecture, with a strong “plug in memory quickly” pitch. 

### Letta (formerly MemGPT)

**What it’s doing:** A platform + API for **stateful agents** with explicit “memory blocks” and an agent object model (system prompt, memory blocks, tools, etc.).   
**Differentiation:** More “agent framework / agent runtime” than just storage—i.e., a structured approach to what goes in context and why (“memory blocks”) and a model-agnostic persisted agent concept. 

### Zep

**What it’s doing:** A “context engineering + agent memory” platform built around a **temporal knowledge graph** and “episodes” / time-aware memory, with research backing (papers evaluating retrieval).   
**Differentiation:** Strong focus on _time_, _structure_, and _relationship-aware retrieval_—not just chunk + embed. Also leans into “GraphRAG + context assembly.” 

### Graphiti (Zep’s framework)

**What it’s doing:** A concrete implementation approach for agent memory using a **temporally aware knowledge graph** (often discussed with Neo4j).   
**Differentiation:** More “how you build it” than “buy our hosted memory”—a framework angle and “real-time updates” / agentic workloads as the target. 

### LangMem (LangChain / LangGraph ecosystem)

**What it’s doing:** An SDK for agents to **extract important info from conversations**, maintain long-term memory, and even **optimize behavior via prompt refinement**—with tight integration into LangGraph storage.   
**Differentiation:** Developer-first primitives and LangGraph-native integration; also explicitly pushes “agents improve over time” via prompt updates, not only retrieval. 

### Cognee

**What it’s doing:** An “open source AI memory engine / knowledge engine” that emphasizes **data ingestion pipelines + graph + vector** and “context curation / ontology mapping / session management.”   
**Differentiation:** Positions as a broader _knowledge infrastructure layer_ (ingestion, modeling, context curation), not only conversation memory. 

### Hyperspell

**What it’s doing:** A memory layer that is very **connectors-first**: connect Slack/Gmail/Notion/Drive, continuously index, and generate context for agents.   
**Differentiation:** “Plug & play” enterprise/workspace context network—less “agent framework,” more “wire your company data into agent memory fast.” 

### Memary

**What it’s doing:** Open-source “agent memory” focused on **human-like memory** concepts and (often) knowledge graphs / “action graphs,” plus tooling/insights into what the agent “remembers.”   
**Differentiation:** More experimental/agent-research vibe; emphasizes introspection/analysis of memories and human-inspired structures. 

### Supermemory

**What it’s doing:** A “universal memory API” that ingests lots of formats (PDF, docs, video, images) and builds a **vector+graph** memory layer / “living knowledge graph” framing.   
**Differentiation:** Broad ingestion + “knowledge graph mental model” positioning, aiming at personalization via a unified memory API.
