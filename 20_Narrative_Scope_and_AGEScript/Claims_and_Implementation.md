# Claims and Implementation — Narrative Scope, AGEScript, and Living World

## Claims

- Story is represented through stateful consequences.
- AGEScript absorbs derailment through rebinding.
- Living-world change is scoped by flags and ticks.

## Implementation steps

1. Define narrative scope records.
2. Build AGEScript schema.
3. Implement preconditions and consequences.
4. Implement plot rebinding.
5. Build faction/background clocks.
6. Add manifestation rules for macro-events.

## Risks and controls

- Risk: AGEScript too complex. Control: visual authoring templates.
- Risk: world change overwhelms play. Control: scope-based manifestation.
- Risk: fake agency. Control: stateful consequences must persist.
