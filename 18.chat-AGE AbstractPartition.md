### III. Critical Constraints for State Coherence
To ensure this simplicity does not reintroduce **Execution-Vulnerable Drift**, the following constraints must be enforced:

#### 1. Deterministic Lazy Instantiation (The "Seed" Rule)
*   **Constraint:** When a player enters a lazily instantiated partition (e.g., a new star system), its properties must be generated **deterministically** based on the `Adventure Seed` + `Partition Coordinates`, **not** by the LLM.
*   **Why:** If the LLM generates the planet details on the fly, Player A and Player B visiting the same system later will see different planets (Drift).
*   **Implementation:**
    ```python
    def instantiate_partition(partition_id, seed):
        # Deterministic procedural generation (Noise functions, tables)
        properties = procedural_generator.generate(seed, partition_id)
        # LLM ONLY describes the pre-generated properties
        return properties
    ```

#### 2. Global Flag Propagation into Lazy Partitions
*   **Constraint:** When a partition is instantiated, it must inherit **Global/Upstream Flags** immediately.
*   **Why:** If a `Galactic War` starts at the Epic level, a lazily generated star system created *after* the war began must reflect blockades, refugee flows, or military presence immediately.
*   **Implementation:**
    ```python
    def on_partition_instantiated(partition):
        global_flags = flag_registry.get_upstream_flags(scope="galactic")
        partition.apply_flags(global_flags) # e.g., set "blockade_active: true"
    ```

#### 3. Voronoi vs. Grid Semantics
*   **Grid (Coordinates):** Used for **Travel & Physics** (Distance, Fuel, Time).
*   **Voronoi (Nodes):** Used for **Jurisdiction & Narrative** (Political Borders, Faction Control, Story Zones).
*   **Benefit:** A player can be at Grid Coordinate (100, 100) but belong to Voronoi Node "Federated Systems." This allows narrative boundaries to differ from spatial boundaries (e.g., smugglers' zones).

#### 4. DomainPartition for AI Assistants
*   **Extension:** This allows the engine to power non-game entities (e.g., a Roleplaying AI Assistant).
*   **Mechanism:** The `DomainPartition` anchors the LLM to a specific context (e.g., "Medical Advice," "Legal Reasoning," "Character Roleplay").
*   **Constraint:** The **Bridge Layer** must enforce **Safety Boundaries** specific to the domain (e.g., Medical Partition cannot give definitive diagnoses, only general info).
