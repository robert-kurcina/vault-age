# Claims and Implementation — Agent Service, MCP, and Professional Extensions

## Claims

- Agent Service is deferred until core systems are mature.
- External action requires explicit authorization.
- Knowledge answers and real-world actions are separate.

## Implementation steps

1. Define action envelope schema.
2. Define permission model.
3. Integrate Role Semantic Contracts.
4. Add tool gateway adapters.
5. Add human confirmation policies.
6. Add audit/export logs.

## Risks and controls

- Risk: premature deployment. Control: defer.
- Risk: unauthorized action. Control: permissions and confirmations.
- Risk: data exposure. Control: tool scopes and logs.
