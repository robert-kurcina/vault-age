## 2. AGEScript Syntax Specification

AGEScript uses a YAML-like structure for readability but compiles to a deterministic JSON bytecode for the Bridge Layer.

### 2.1 Scene Header (Plot & Context)
Defines the narrative constraints and state anchors.
```yaml
@Scene: tavern_brawl_01
@Plot: polti_03  # "Crime Pursued by Vengeance"
@Phase: crisis   # Must resolve this scene to advance Plot Phase
@Duration: 5     # Turns (Spotlights)
@Location: locus_blackstone_tavern
@Anchors:        # State to load into LLM context (Vector DB + Relational)
  - flag: tavern_hostility
  - flag: player_reputation
  - item: player_weapon_durability
  - npc: barmaid_elara_status
```

### 2.2 Spotlight (Decision Point)
Defines player inputs. Supports **Troupe Aggregation** to prevent branching explosion.
```yaml
@Spotlight: 1
@Type: Troupe_Choice  # All players submit before resolution
@Prompt: "The guards draw steel. How does the troupe respond?"
@Inputs:
  - id: fight
    label: "Engage the guards"
    weight: 1         # For majority voting logic
  - id: distract
    label: "Create a diversion"
    weight: 1
  - id: flee
    label: "Retreat to the alley"
    weight: 1
@Constraints:          # LLM Narrative Guardrails
  - trope: chekhovs_gun  # Must reference item established earlier
  - tone: tense
```

### 2.3 Resolution Logic (The Branching Collapser)
Instead of branching per player, logic aggregates inputs into **3-5 Outcomes Max**.
```yaml
@Resolution:
  - condition: "count(fight) >= 3"
    outcome: violent_clash
  - condition: "count(distract) >= 2 OR (count(fight) == 1 AND count(distract) == 1)"
    outcome: chaotic_escape
  - condition: "count(flee) >= 3"
    outcome: cowardly_retreat
  - default: outcome: stalemate
```

### 2.4 Outcome (State Mutation & Narrative Hooks)
Defines database updates and LLM anchors.
```yaml
@Outcome: violent_clash
@StateMutations:       # Bridge Layer API Calls
  - flag: tavern_hostility += 50
  - flag: player_reputation -= 10
  - item: player_weapon_durability -= 10
  - npc: barmaid_elara_status = hostile
@PlotProgress:         # Plot Engine Integration
  - phase: crisis_resolved
  - next_phase: fallout
@NarrativeHooks:       # LLM Anchors (Prevents Drift)
  - fact: "Guards drew first blood"
  - fact: "Table overturned during scuffle"
  - fact: "Elara witnessed the violence"
@TextSeed: "Steel rings against stone as the troupe commits to violence..."
```

## 3. Troupe Play Example: 5 Players, 1 Scene

In Ink, 5 players choosing 3 options = $3^5 = 243$ branches.
In AGEScript, 5 players choosing 3 options = **3 Aggregated Outcomes**.

```yaml
@Scene: dragon_negotiation
@Duration: 3
@Spotlight: 1
@Type: Troupe_Role_Assign  # Players pick roles, not just choices
@Roles:
  - speaker   # Makes the Diplomacy roll
  - guardian  # Makes the Defense roll
  - scout     # Makes the Perception roll
@Resolution:
  - condition: "role.speaker.success AND role.guardian.success"
    outcome: peaceful_accord
  - condition: "role.speaker.fail AND role.guardian.success"
    outcome: tense_standoff
  - condition: "role.guardian.fail"
    outcome: dragon_attack
@Outcome: dragon_attack
@StateMutations:
  - flag: locus_danger_level = critical
  - hp: party -= 20%
@NarrativeHooks:
  - fact: "Dragon breathed fire immediately"
  - fact: "Guardian shielded the Speaker"
```

**Why this works:**
1.  **Roles reduce complexity:** Players don't all do everything; they specialize.
2.  **Aggregation collapses branches:** Outcomes depend on *success/failure patterns*, not individual choice permutations.
3.  **State Mutations are explicit:** The Bridge Layer validates these before committing.

## 4. Integration with Architecture

### 4.1 Compilation Pipeline
```
AGEScript (.age)
    ↓
Compiler (Validates Syntax + Plot Constraints)
    ↓
Bytecode (.json) → Stored in Content DB
    ↓
Bridge Layer (Loads at Runtime)
    ↓
Executes Spotlight → Collects Inputs → Resolves Outcome → Commits State
    ↓
LLM (Generates Prose using NarrativeHooks + StateMutations)
```

### 4.2 LLM Anchoring Mechanism
AGEScript provides the **Structured Context** the LLM needs to avoid drift.
```python
# Bridge Layer constructs LLM Prompt
prompt = f"""
SCENE CONTEXT: {scene.text_seed}
CURRENT STATE: {state_snapshot}
NARRATIVE HOOKS (MUST INCLUDE):
{outcome.narrative_hooks}
TROPE CONSTRAINTS:
{spotlight.constraints}

Generate the resolution prose for outcome: {outcome.id}
"""
```
**Result:** The LLM cannot hallucinate that "Elara smiled warmly" because `@Outcome` set `npc: barmaid_elara_status = hostile`.

### 4.3 Database Update (Bridge Layer)
AGEScript `@StateMutations` map directly to your OCC-protected API.
```python
# Pseudo-code for Bridge Layer Execution
for mutation in outcome.state_mutations:
    if mutation.type == 'flag':
        flag_registry.update(mutation.key, mutation.operator, mutation.value, client_version)
    elif mutation.type == 'item':
        inventory_engine.modify_durability(mutation.item_id, mutation.value)
```
If OCC fails (concurrent write), the Bridge Layer retries or triggers a **Conflict Spotlight** (e.g., "Your action was interrupted!").

## 7. Example: 20-Turn Campaign Arc (Condensed)

```yaml
@Arc: shadow_over_sondgara
@Chapters:
  - id: ch1_investigation
    scenes: [tavern_brawl, ruin_exploration, council_meeting]
    plot_goal: discover_conspiracy
  - id: ch2_confrontation
    scenes: [bridge_ambush, tower_assault]
    plot_goal: defeat_arch_villain
@GlobalState:
  - flag: conspiracy_exposed (default: false)
  - flag: arch_villain_alive (default: true)
@EndingConditions:
  - if: conspiracy_exposed AND NOT arch_villain_alive
    outcome: heroic_victory
  - if: NOT conspiracy_exposed
    outcome: tragic_ignorance
```

**Why this helps:**
-   **Macro-Structure:** Defines the Arc without writing every knot.
-   **State-Driven Endings:** Endings depend on flags, not specific path history.
-   **LLM Guidance:** The LLM knows the *goal* of the Chapter (`discover_conspiracy`) and can tailor scene descriptions to hint at it.
