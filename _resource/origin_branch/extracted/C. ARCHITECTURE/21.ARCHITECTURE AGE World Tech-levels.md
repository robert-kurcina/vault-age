### III. Bridge Layer TL Enforcement (Drift Prevention)

The Bridge Layer **validates all player actions and LLM output** against TL constraints:

```python
def validate_tl_compliance(action, tl_vector, spatial_node_id):
violations = []

# 1. Check item availability
if action.item_requested:
    if action.item_requested in tl_vector.constraints.prohibited_items:
        violations.append(f"Item {action.item_requested} unavailable at TL {tl_vector.baseline}")

# 2. Check concept availability
if action.concept_invoked:
    if action.concept_invoked in tl_vector.constraints.prohibited_concepts:
        violations.append(f"Concept {action.concept_invoked} unknown at TL {tl_vector.baseline}")

# 3. Check jump tech flags (temporary access)
if action.item_requested in tl_vector.jump_tech_flags:
    flag = tl_vector.jump_tech_flags[action.item_requested]
    if current_tick > flag.expiry_tick:
        violations.append(f"Jump tech {action.item_requested} has decayed")

# 4. Reject before LLM sees action
if violations:
    return Rejection(
        error="tl_violation",
        details=violations,
        suggestion=get_tl_appropriate_alternative(action, tl_vector)
    )

return Accept(action)
```
