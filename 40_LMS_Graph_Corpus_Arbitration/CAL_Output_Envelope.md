# CAL Output Envelope

## Purpose

CAL answers should not merely state a conclusion. They should expose how the conclusion was reached and where human authority remains necessary.

## Standard fields

- User question.
- Interpreted query.
- Active corpus.
- Corpus partition searched.
- Adjacent partitions considered.
- Sources retrieved.
- Authority tier for each source.
- Version or edition.
- Jurisdiction, table scope, or rules scope.
- Confidence or weight.
- Conflicts found.
- Primary answer.
- Alternative readings.
- Human decision point.
- Deep-dive paths.
- Unsupported or excluded material.
- Table-specific override, if any.
- Arbitration log identifier.

## Quick ruling form

Quick rulings are for play. They should include the answer, active source tier, table-policy effect, and whether a human decision point exists.

## Detailed ruling form

Detailed rulings are for audit, authoring, dispute review, QA, and corpus improvement. They should include the full envelope.

## Rule

The answer envelope is part of the product. It makes CAL auditable, bounded, and reviewable without pretending that the system is omniscient.
