# Large Model System Graph [LMS-Graph] Recursive Ingestion Model

## Local definitions

Large Model System [LMS] means the wider model, retrieval, graph, tool, agent, and workspace system around one or more LLMs.

Large Model System Graph [LMS-Graph] means the maintained graph/relational corpus substrate.

## Ingestion principle

LMS-Graph begins from authoritative roots and expands through citations, links, dependencies, examples, errata, supplements, rulings, versions, and adjacent concepts.

## Exploration pattern

The default pattern is breadth-first, then focused depth.

Breadth-first ingestion establishes the baseline map of the corpus. Focused depth follows user questions, author needs, monitor-agent priorities, conflict queues, and high-value gaps.

## Storage pattern

Graph storage represents relationships: cites, overrides, supplements, conflicts, defines, excepts, depends-on, applies-to, supersedes, table-overrides, and equivalent-to.

Relational storage represents repeatable facts: dates, versions, editions, thresholds, classes, parameters, requirement fields, status, source type, source owner, and license state.

## Resource control

Unbounded ingestion is not literal infinity. It is expansion subject to configured limits: cost, storage, licensing, relevance, depth, priority, freshness, and stopping criteria.

## Monitor agent

The monitor agent tracks coverage density, citation resolution, freshness, conflict queue depth, query failure rate, user correction rate, referee override rate, and source-version drift.

## Human decision points

When authority weights are close, sources conflict, or scope is ambiguous, the system flags a human decision point rather than flattening the result into false certainty.
