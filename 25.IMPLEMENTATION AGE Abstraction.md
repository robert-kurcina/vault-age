public class DomainPartition extends AbstractPartition {  
  
    private final Map<Class<?>, PartitionFacet<?>> facets = new LinkedHashMap<>();  
  
    private final TickPolicy tickPolicy;  
    private final BridgePolicy bridgePolicy;  
    private final ValidationPolicy validationPolicy;  
    private final DeterminismPolicy determinismPolicy;  
  
    private final Deque<Event> inbound = new ArrayDeque<>();  
    private final Deque<Event> outbound = new ArrayDeque<>();  
  
    private Proposal pending;  
    private final List<Commit> history = new ArrayList<>();  
  
    public DomainPartition(  
            PartitionId id,  
            String name,  
            TickPolicy tickPolicy,  
            BridgePolicy bridgePolicy,  
            ValidationPolicy validationPolicy,  
            DeterminismPolicy determinismPolicy  
    ) {  
        super(id, name);  
        this.tickPolicy = tickPolicy;  
        this.bridgePolicy = bridgePolicy;  
        this.validationPolicy = validationPolicy;  
        this.determinismPolicy = determinismPolicy;  
    }  
  
    public <T> void addFacet(PartitionFacet<T> facet) {  
        facets.put(facet.token(), facet);  
    }  
  
    public boolean hasFacet(Class<?> token) {  
        return facets.containsKey(token);  
    }  
  
    @SuppressWarnings("unchecked")  
    public <T> Optional<PartitionFacet<T>> facet(Class<T> token) {  
        return Optional.ofNullable((PartitionFacet<T>) facets.get(token));  
    }  
  
    public Collection<PartitionFacet<?>> facets() {  
        return Collections.unmodifiableCollection(facets.values());  
    }  
  
    public DeterminismPolicy determinism() { return determinismPolicy; }  
    public TickPolicy tick() { return tickPolicy; }  
    public BridgePolicy bridge() { return bridgePolicy; }  
  
    @Override  
    public void receive(Event e) {  
        inbound.addLast(e);  
    }  
  
    @Override  
    public List<Event> drainOutboundEvents() {  
        List<Event> out = new ArrayList<>(outbound);  
        outbound.clear();  
        return out;  
    }  
  
    @Override  
    public void propose(Proposal proposal) {  
        validationPolicy.validateProposal(this, proposal);  
        this.pending = proposal;  
    }  
  
    @Override  
    public Commit commit(Map<String, String> meta) {  
        if (pending == null) throw new IllegalStateException("No pending proposal to commit.");  
        Commit c = new Commit(CommitId.random(), pending.changes(), Instant.now(), meta);  
        history.add(c);  
        pending = null;  
        return c;  
    }  
  
    @Override  
    public void rollback(CommitId commitId) {  
        // skeleton: you’d implement snapshotting or inverse ops here  
        history.removeIf(c -> c.id().equals(commitId));  
    }  
  
    // Example: emit outbound events (bridges decide cross-partition eligibility)  
    public void emit(Event e) {  
        outbound.addLast(e);  
    }  
}

## Design summary

### Key ideas

- Every `DomainPartition` can optionally have an owning `TroupeId`.

- **BridgePolicy** is the semantic firewall:
    
    - Deny cross-troupe traffic by default
        
    - Allow only “ripple” events to/from global partitions
        
    - Transform outbound events into redacted “news/rumor” events when necessary

## 4) Topology graph: partitions + directed links + router

```java
import java.util.*;  
  
public enum LinkKind {  
    PARENT_CHILD,  
    ADJACENT,  
    OVERLAY,     // concern overlays (economy, law) across a spatial scope  
    WORLD_BUS    // explicit: troupe <-> global exchange hub  
}  
  
public record PartitionLink(  
        PartitionId from,  
        PartitionId to,  
        LinkKind kind,  
        BridgePolicy policy  
) {}  
  
public final class PartitionTopology {  
    private final Map<PartitionId, DomainPartition> partitions = new HashMap<>();  
    private final Map<PartitionId, List<PartitionLink>> outLinks = new HashMap<>();  
  
    public void addPartition(DomainPartition p) {  
        partitions.put(p.id(), p);  
        outLinks.computeIfAbsent(p.id(), _k -> new ArrayList<>());  
    }  
  
    public void link(PartitionId from, PartitionId to, LinkKind kind, BridgePolicy policy) {  
        outLinks.computeIfAbsent(from, _k -> new ArrayList<>())  
                .add(new PartitionLink(from, to, kind, policy));  
    }  
  
    public DomainPartition get(PartitionId id) {  
        DomainPartition p = partitions.get(id);  
        if (p == null) throw new IllegalArgumentException("Unknown partition: " + id);  
        return p;  
    }  
  
    /**  
     * Route all outbound events for a single partition along its outgoing links.  
     * You can call this per tick, per commit, or in a main loop.  
     */  
    public void routeOutbound(PartitionId fromId) {  
        DomainPartition from = get(fromId);  
        List<Event> events = from.drainOutboundEvents();  
        List<PartitionLink> links = outLinks.getOrDefault(fromId, List.of());  
  
        for (Event e : events) {  
            for (PartitionLink link : links) {  
                DomainPartition to = get(link.to());  
                BridgePolicy policy = link.policy();  
  
                if (!policy.canSend(e, from, to)) continue;  
  
                Event transformed = policy.transform(e, from, to);  
                to.receive(transformed);  
            }  
        }  
    }  
}
```

This is your **anti-recursion enforcement point**: modules don’t call other partitions; they only `emit()`. The router decides what crosses.

## 5) Bridge policies that enforce troupe isolation + ripples via global hubs

### 5.1 Default: deny cross-troupe unless explicitly allowed

```java
import java.util.*;  
  
public final class DenyCrossTroupePolicy implements BridgePolicy {  
    @Override  
    public boolean canSend(Event e, DomainPartition from, DomainPartition to) {  
        Optional<TroupeId> a = from.troupeOwner();  
        Optional<TroupeId> b = to.troupeOwner();  
        if (a.isPresent() && b.isPresent() && !a.get().equals(b.get())) return false;  
        return true;  
    }  
  
    @Override  
    public Event transform(Event e, DomainPartition from, DomainPartition to) {  
        return e;  
    }  
}
```

### 5.2 World bus: troupe↔global with redaction rules

This is the “insurmountable distance” implementation: you only wire troupe partitions to a shared global hub (`Spatial:World`, `Narrative:Epic`, or similar). No troupe↔troupe edges exist.

```java
import java.util.*;  
  
public final class WorldRippleBusPolicy implements BridgePolicy {  
  
    // Only allow RIPPLE events to traverse troupe <-> global.  
    @Override  
    public boolean canSend(Event e, DomainPartition from, DomainPartition to) {  
        // must be RIPPLE to leave a troupe partition  
        if (e.clazz() != EventClass.RIPPLE) return false;  
  
        boolean fromTroupe = from.troupeOwner().isPresent();  
        boolean toTroupe = to.troupeOwner().isPresent();  
  
        // Only allow troupe -> global or global -> troupe  
        if (fromTroupe == toTroupe) return false;  
  
        // Require the global side be sufficiently coarse (World or Region usually)  
        DomainPartition globalSide = from.isGlobal() ? from : to;  
        SpatialRank sr = globalSide.spatialRank().orElse(SpatialRank.SUBMAP);  
        if (sr != SpatialRank.WORLD && sr != SpatialRank.REGION) return false;  
  
        return true;  
    }  
  
    @Override  
    public Event transform(Event e, DomainPartition from, DomainPartition to) {  
        boolean crossingIntoOtherTroupe = from.isGlobal() && to.troupeOwner().isPresent();  
        boolean leavingTroupe = from.troupeOwner().isPresent() && to.isGlobal();  
  
        if (leavingTroupe) {  
            // troupe -> global: redact + aggregate at intake  
            return redactToWorldNews(e, from);  
        }  
        if (crossingIntoOtherTroupe) {  
            // global -> troupe: deliver as rumor/news, still redacted  
            return deliverAsRumor(e, to);  
        }  
        return e;  
    }  
  
    private Event redactToWorldNews(Event e, DomainPartition from) {  
        Map<String,Object> p = new HashMap<>(e.payload());  
  
        // Example redactions: strip proper names, precise coordinates, internal IDs  
        p.remove("npcName");  
        p.remove("exactLocation");  
        p.remove("internalId");  
  
        // Add provenance as coarse metadata  
        p.put("sourceTroupe", from.troupeOwner().map(t -> t.value().toString()).orElse("unknown"));  
        p.put("confidence", 0.6); // illustrative  
  
        return new Event("WORLD_NEWS:" + e.type(), EventClass.RIPPLE, p, e.at());  
    }  
  
    private Event deliverAsRumor(Event e, DomainPartition to) {  
        Map<String,Object> p = new HashMap<>(e.payload());  
        p.put("deliveryMode", "RUMOR");  
        p.put("intendedTroupe", to.troupeOwner().map(t -> t.value().toString()).orElse("global"));  
        return new Event("RUMOR:" + e.type(), EventClass.RIPPLE, p, e.at());  
    }  
}
```

**Effect:** a troupe can “move the world,” but other troupes only see _coarse echoes_, not direct narrative entanglement.

## 6) Wiring example: two troupes isolated, connected only via World/Epic hub

```java
// Create IDs  
TroupeId troupeA = TroupeId.random();  
TroupeId troupeB = TroupeId.random();  
  
// Global hub: SpatialPartition:World + NarrativePartition:Epic  
DomainPartition worldEpic = new DomainPartition(  
        PartitionId.random(),  
        "World/Epic Hub",  
        Optional.empty(),  
        () -> 2_592_000_000L,     // ~30 days in ms (illustrative)  
        new DenyCrossTroupePolicy(),  
        (p, prop) -> {},  
        () -> true  
);  
worldEpic.addFacet(new BasicSpatialFacet<>(World.class));  
worldEpic.addFacet(new BasicNarrativeFacet<>(Epic.class));  
  
// Troupe A scene: SpatialPartition:Locus + NarrativePartition:Scene  
DomainPartition troupeAScene = new DomainPartition(  
        PartitionId.random(),  
        "TroupeA Scene",  
        Optional.of(troupeA),  
        () -> 60_000L,            // 1 minute tick (illustrative)  
        new DenyCrossTroupePolicy(),  
        (p, prop) -> {},  
        () -> false               // maybe soft mode for gameplay  
);  
troupeAScene.addFacet(new BasicSpatialFacet<>(Locus.class));  
troupeAScene.addFacet(new BasicNarrativeFacet<>(Scene.class));  
  
// Troupe B arc: SpatialPartition:Region + NarrativePartition:Arc  
DomainPartition troupeBArc = new DomainPartition(  
        PartitionId.random(),  
        "TroupeB Arc",  
        Optional.of(troupeB),  
        () -> 86_400_000L,        // 1 day tick (illustrative)  
        new DenyCrossTroupePolicy(),  
        (p, prop) -> {},  
        () -> false  
);  
troupeBArc.addFacet(new BasicSpatialFacet<>(Region.class));  
troupeBArc.addFacet(new BasicNarrativeFacet<>(Arc.class));  
  
// Build topology  
PartitionTopology topo = new PartitionTopology();  
topo.addPartition(worldEpic);  
topo.addPartition(troupeAScene);  
topo.addPartition(troupeBArc);  
  
// World bus links (no troupe<->troupe links exist)  
BridgePolicy worldBus = new WorldRippleBusPolicy();  
topo.link(troupeAScene.id(), worldEpic.id(), LinkKind.WORLD_BUS, worldBus);  
topo.link(worldEpic.id(), troupeBArc.id(), LinkKind.WORLD_BUS, worldBus);  
  
// Troupe A emits a ripple  
Map<String,Object> payload = new HashMap<>();  
payload.put("npcName", "CaptiveB");  
payload.put("exactLocation", "Village-13 / Hut-2");  
payload.put("event", "CAPTIVE_FREED");  
payload.put("impact", "Faction retaliation likely");  
troupeAScene.emit(Event.ripple("CAPTIVE_FREED", payload));  
  
// Route A -> World  
topo.routeOutbound(troupeAScene.id());  
  
// Route World -> B (in a real engine you’d do this on world tick or after aggregation)  
topo.routeOutbound(worldEpic.id());  
  
// Now troupeBArc has received a redacted rumor/news event in its inbound queue
```

This matches your earlier dynamicism examples:

- troupe A does something locally
    
- a **background world partition** absorbs it
    
- troupe B can later perceive it as a **rumor / consequence**, not as a shared scene

## Where ConcernPartition:* fits in this topology

Concern overlays should usually be:

- **Overlay links** within the same troupe (local “law layer” on top of a scene)
    
- Or overlay links to **global concern partitions** (world economy), but still controlled via bus policies
    

You can model that as `LinkKind.OVERLAY` with a policy like:

- “Concern overlay may read state summaries but cannot mutate spatial state directly”
    
- “Concern overlay proposals must be committed by the spatial partition’s controller”
    

(That’s how you keep LLM-heavy concern reasoning from directly rewriting authoritative world-state.)
